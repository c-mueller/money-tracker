# Plan 007: Internationalization (i18n) — German/English

## Status: Implemented

## Summary
The web UI now supports German and English. The language is set instance-wide via config (default: `de`), but can be overridden per request via the `Accept-Language` header. API error messages and domain validations remain in English.

## Approach
Simple map-based translation without an external framework. Translations as Go maps in `internal/i18n/`. Template function `{{t "key"}}` for all UI strings.

## New Files

- `internal/i18n/i18n.go` — Core: Locale type, Bundle, T() function, ParseLocale, ParseAcceptLanguage
- `internal/i18n/de.go` — German translations (~95 keys)
- `internal/i18n/en.go` — English translations
- `internal/i18n/frequencies.go` — Frequency display names per locale (Daily/Täglich etc.)
- `internal/i18n/i18n_test.go` — Tests for locale parsing, Accept-Language, translation lookup
- `internal/middleware/locale.go` — Accept-Language middleware

## Modified Files

- `internal/config/config.go` — New `Language` field (default: `"de"`, ENV: `MONEY_TRACKER_LANGUAGE`)
- `internal/api/server.go` — New fields `i18nBundle`, `defaultLocale`; `NewServer()` takes `language string`
- `internal/api/router.go` — Locale middleware for web routes, bundle passed to renderer
- `internal/api/template.go` — Bundle integrated, template funcs `t` (translate), `tf` (frequency), `formatDateISO` (for input[type=date]); `formatMoney`/`formatDate` locale-aware (DE: `1.234,50` / `02.01.2006`, EN: `1,234.50` / `01/02/2006`)
- `internal/api/web_handler.go` — `pageData.Lang` field, titles as i18n keys
- `cmd/money-tracker/cmd/serve.go` — Language config passed to NewServer
- `tests/integration/testutil.go` — NewServer call updated
- All HTML templates — Strings → `{{t "key"}}`, frequencies → `{{tf ...}}`, `<html lang="{{.Lang}}">`
- `web/static/app.js` — Uses `window.i18n` for copy/error strings
- Layout template — Injects `window.i18n` object via script tag

## Configuration

```bash
# Server default German (standard)
MONEY_TRACKER_LANGUAGE=de make run-dev

# Server default English
MONEY_TRACKER_LANGUAGE=en make run-dev

# Override per request
curl -H "Accept-Language: en" localhost:8080
```
